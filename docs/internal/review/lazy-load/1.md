# P0：React `event-bridge` 遇到 kebab-case 事件名会生成非法 JavaScript

当前 `createEventBindings()` 使用事件名直接拼接局部变量：

```ts
localName: `eventHandler${eventName}`
```

生成代码又把该值当 JavaScript 标识符使用。事件名为 `value-change` 时，会生成 `eventHandlervalue-change`，这是非法语法。

## 修改 `packages/web-c/output-react-wrapper/src/generateReactWrapper.ts`

替换 `createEventBindings`：

```ts
function createEventBindings(eventNames: string[]): EventBinding[] {
  return eventNames.map((eventName, index) => ({
    eventName,
    sourceName: toReactEventProp(eventName),
    localName: `eventHandler${index}`,
  }))
}
```

## 修改 `packages/web-c/component-dts/src/generateReactDts.ts`

事件属性也应该经过 `safePropertyName()`，避免 `update:value` 一类事件名生成非法声明：

```ts
for (const [name, event] of Object.entries(component.events)) {
  const propName = toReactEventProp(name)
  const detailType = event.detail ? formatDetailType(event.detail) : 'unknown'

  lines.push(
    `  ${safePropertyName(propName)}?: (event: CustomEvent<${detailType}>) => void`,
  )
}
```

# P0：自定义 `CustomElementRegistry` 会把完整组件泄漏注册到全局 registry

生成的 loader 对外支持传入自定义 registry。
但 lazy entry 会导入原始组件源码，而源码中的 `defineElement()` 仍然会对全局 `customElements` 执行注册。

因此：

```ts
defineCustomElements({
  registry: scopedRegistry,
})
```

注册 Proxy Element 到 `scopedRegistry` 后，组件首次懒加载时，完整实现仍可能被注册到全局 `customElements`。

## 推荐修改：标记由 lazy loader 托管的 tag

### 替换 `packages/web-c/web-c-runtime/src/bootstrapLazy.ts`

```ts
import { createLazyElementClass } from './lazy-element'

import type { BootstrapLazyOptions, ZeusLazyComponentMeta } from './types'

const ZEUS_LAZY_MANAGED_TAGS = Symbol.for('zeus.web-c.lazy-managed-tags')

function getLazyManagedTags(): Set<string> {
  const globalObject = globalThis as typeof globalThis & {
    [ZEUS_LAZY_MANAGED_TAGS]?: Set<string>
  }

  globalObject[ZEUS_LAZY_MANAGED_TAGS] ??= new Set<string>()

  return globalObject[ZEUS_LAZY_MANAGED_TAGS]
}

export function bootstrapLazy(
  components: ZeusLazyComponentMeta[],
  options: BootstrapLazyOptions = {},
): void {
  const registry =
    options.registry ??
    (typeof customElements !== 'undefined' ? customElements : undefined)

  if (!registry) {
    return
  }

  const managedTags = getLazyManagedTags()

  for (const meta of components) {
    managedTags.add(meta.tagName)
  }

  for (const meta of components) {
    if (registry.get(meta.tagName)) {
      continue
    }

    const LazyElement = createLazyElementClass(meta)

    registry.define(meta.tagName, LazyElement)
  }
}
```

### 修改 `packages/core/runtime-dom/src/defineElement.ts`

在 `ZEUS_ELEMENT_DEFINITION` 附近新增：

```ts
const ZEUS_LAZY_MANAGED_TAGS = Symbol.for('zeus.web-c.lazy-managed-tags')

function isLazyManagedTag(tagName: string): boolean {
  const globalObject = globalThis as typeof globalThis & {
    [ZEUS_LAZY_MANAGED_TAGS]?: Set<string>
  }

  return globalObject[ZEUS_LAZY_MANAGED_TAGS]?.has(tagName) ?? false
}
```

将注册逻辑：

```ts
if (!customElements.get(tagName)) {
  customElements.define(tagName, ZeusElement)
}
```

替换为：

```ts
if (!isLazyManagedTag(tagName) && !customElements.get(tagName)) {
  customElements.define(tagName, ZeusElement)
}
```

# P0：默认 `minimal` wrapper 与 React 18 支持声明冲突

`@zeus-js/output-react-wrapper` 当前声明支持 React 18 和 React 19。
但默认 `minimal` wrapper 只把 props 传给 Custom Element，不进行 property sync，也不桥接事件。

React 官方说明，完整 Custom Elements 支持以及将匹配的 props 赋值为 properties，是 React 19 新增的能力；更早版本会把未知 props 当作 attributes。([React][1])

因此 React 18 下，数组、对象、函数 props 和 CustomEvent 不能依赖默认 minimal wrapper 正常工作。

## 推荐决策

既然已经确定 wrapper 默认极简，建议明确以 React 19 为默认支持目标。

### 修改 `packages/web-c/output-react-wrapper/package.json`

```json
{
  "peerDependencies": {
    "react": ">=19"
  }
}
```

### 修改 `packages/web-c/output-react-wrapper/src/types.ts`

```ts
/**
 * minimal:
 *   Default. Requires React 19+.
 *   React wrapper only renders the custom element tag.
 *   No useEffect prop sync, no event listeners.
 *
 * event-bridge:
 *   Compatibility mode for React 18 or applications that require explicit
 *   CustomEvent bridging and property assignment.
 */
wrapper?: ReactWrapperMode

/**
 * Named slot strategy for both minimal and event-bridge modes.
 *
 * props:
 *   <ZCard header={<div />} />
 *
 * none:
 *   only children/default slot
 *
 * @default 'props'
 */
namedSlots?: 'props' | 'none'
```

需要继续支持 React 18 时，应要求组件库构建配置显式使用：

```ts
reactWrapper({
  wrapper: 'event-bridge',
})
```

# P1：lazy entry 可以进一步简化，不需要生成 class 和空方法

当前 entry 生成了 class、空 `attributeChanged()` 和空 `render()`。

## 替换 `packages/web-c/output-wc/src/generateLazyEntry.ts` 的生成主体

```ts
export function generateLazyEntry(options: GenerateLazyEntryOptions): string {
  const { component, outPath, sourceImport } = options
  const source = sourceImport ?? toRelativeImport(component.source, outPath)

  return [
    `import { mountElementDefinition } from "@zeus-js/runtime-dom";`,
    `import { ${component.exportName} } from ${JSON.stringify(source)};`,
    '',
    `export function createComponent(hostRef) {`,
    `  let mounted;`,
    `  const mountState = {};`,
    '',
    `  return {`,
    `    connected() {`,
    `      if (mounted) return;`,
    '',
    `      mounted = mountElementDefinition(`,
    `        ${component.exportName},`,
    `        hostRef.host,`,
    `        hostRef.values,`,
    `        mountState,`,
    `      );`,
    `    },`,
    '',
    `    disconnected() {`,
    `      mounted?.dispose();`,
    `      mounted = undefined;`,
    `    },`,
    '',
    `    propertyChanged(name, oldValue, newValue) {`,
    `      mounted?.propertyChanged(name, oldValue, newValue);`,
    `    },`,
    `  };`,
    `}`,
    '',
    `export default { createComponent };`,
    '',
  ].join('\n')
}
```

生成产物会从：

```txt
class + constructor + connected + disconnected
+ propertyChanged + attributeChanged + render
```

缩减为：

```txt
createComponent()
  -> connected
  -> disconnected
  -> propertyChanged
```

# P1：`namedSlots: 'none'` 与 React `.d.ts` 不一致

运行时代码会读取 `normalized.namedSlots`，但声明文件生成时没有传入该配置。

`generateReactDts()` 当前无条件为全部 named slots 生成 props，因此 `namedSlots: 'none'` 时类型仍然声称这些 props 可用。

## 修改 `packages/web-c/component-dts/src/generateReactDts.ts`

```ts
export interface GenerateReactDtsOptions {
  namedSlots?: 'props' | 'none'
}

export function generateReactDts(
  manifest: ComponentManifest,
  options: GenerateReactDtsOptions = {},
): string {
  const lines: string[] = []

  lines.push('/* eslint-disable */')
  lines.push('// Generated by @zeus-js/component-dts.')
  lines.push('')
  lines.push(`import type * as React from 'react'`)
  lines.push('')

  for (const component of manifest.components) {
    lines.push(generateReactComponentDts(component, options))
    lines.push('')
  }

  return lines.join('\n')
}

function generateReactComponentDts(
  component: ComponentRecord,
  options: GenerateReactDtsOptions,
): string {
  // 原有内容保持不变

  if (options.namedSlots !== 'none') {
    for (const name of Object.keys(component.slots)) {
      if (name === 'default') continue

      lines.push(`  ${safePropertyName(name)}?: React.ReactNode`)
    }
  }

  // 原有内容保持不变
}
```

## 修改 `packages/web-c/output-react-wrapper/src/index.ts`

```ts

  namedSlots: normalized.namedSlots,
}),
```
