# 需要修的问题

## P1：`onClick={props.onClick}` 会变成潜在运行时错误

这次 snapshot 里能看到一个变化：

```diff
- _bindEvent(_el$3, "click", props.onClick);
+ _bindEvent(_el$3, "click", _event$ => props.onClick(_event$));
```

问题是，如果组件 props 是：

```tsx
type Props = {
  onClick?: (event: MouseEvent) => void
}

function Button(props: Props) {
  return <button onClick={props.onClick}>Click</button>
}
```

当前会编译成：

```ts
_bindEvent(_el$, 'click', _event$ => props.onClick(_event$))
```

只要用户不传 `onClick`，点击时就会报：

```txt
TypeError: props.onClick is not a function
```

这和 JSX 事件语义不一致。`onClick={undefined}` 应该是不绑定 / no-op，而不是点击时报错。

---

# 推荐修法

最简单、最稳的方案：**所有 member expression 事件 handler 都生成 optional call**。

也就是普通 member：

```tsx
<button onClick={theme.toggle} />
```

编译成：

```ts
_event$ => theme.toggle?.(_event$)
```

可选 member：

```tsx
<button onClick={maybe?.toggle} />
```

编译成：

```ts
_event$ => maybe?.toggle?.(_event$)
```

这样：

```txt
1. theme.toggle 存在时正常调用，并保留 this = theme。
2. props.onClick 不存在时不报错。
3. maybe?.toggle 不存在时不报错。
```

代码草案：

```ts
function normalizeEventHandler(
  handler: t.Expression,
  context: CompilerContext,
): t.Expression {
  if (t.isMemberExpression(handler) || t.isOptionalMemberExpression(handler)) {
    const event = context.uid('event$')

    return t.arrowFunctionExpression(
      [t.identifier(event.name)],
      t.optionalCallExpression(
        t.cloneNode(handler),
        [t.identifier(event.name)],
        true,
      ),
    )
  }

  return handler
}
```

然后测试改成：

```ts
it('wraps member expression event handlers with optional call', async () => {
  const code = `
    const App = () => {
      const theme = state({
        mode: 'light',
        toggle() {
          this.mode = this.mode === 'light' ? 'dark' : 'light'
        },
      })

      return <button onClick={theme.toggle}>Toggle</button>
    }
  `

  expect(await compile(code)).toContain(
    '_bindEvent(_el$, "click", _event$ => theme.toggle?.(_event$));',
  )
})
```

再补一个最关键的 props 测试：

```ts
it('does not throw for optional member event props', async () => {
  const code = `
    type Props = {
      onClick?: (event: MouseEvent) => void
    }

    const App = (props: Props) => {
      return <button onClick={props.onClick}>Click</button>
    }
  `

  expect(await compile(code)).toContain(
    '_bindEvent(_el$, "click", _event$ => props.onClick?.(_event$));',
  )
})
```

这个测试比 `maybe?.toggle` 更重要，因为组件事件透传非常常见。
