# 做得好的点

## 1. delegated events 语义明显增强

现在 `events.ts` 里新增了：

```ts id="infcxu"
const nonBubblingEventMap = {
  focus: 'focusin',
  blur: 'focusout',
}
```

`delegateEvents(['focus'])` 会实际监听 `focusin`，`blur` 会实际监听 `focusout`。

派发时又通过：

```ts id="uzgwjp"
if (name === 'focusin') return 'focus'
if (name === 'focusout') return 'blur'
```

把真实事件名映射回用户绑定的 `focus/blur`。

这个设计适合事件委托。

---

## 2. `event.currentTarget` 语义补上了

现在派发时会从 `event.target` 往上冒泡查 `__zeusEvents`，找到 handler 后用 `callDelegatedHandler(el, handler, event)` 调用。

`callDelegatedHandler` 会临时把 `event.currentTarget` 改成当前命中的元素，然后 finally 恢复。

对应测试也补了：

```ts id="iufltg"
expect(event.currentTarget).toBe(input)
```

以及父子冒泡时 currentTarget 分别是 child、parent。

这块是 runtime 语义上的重要补强。

---

## 3. `resetDelegatedEvents()` 对测试环境有价值

现在新增了：

```ts id="x6fauu"
export function resetDelegatedEvents(): void {
  for (const entry of delegatedListeners) {
    document.removeEventListener(entry.delegatedName, dispatchDelegatedEvent)
  }
  delegatedEvents.clear()
  delegatedListeners.length = 0
}
```

并且在测试 `afterEach` 里调用，避免 document 全局事件监听跨测试污染。

这对 runtime 测试稳定性很有帮助。

---

## 4. render dispose 语义更明确

`render()` 现在返回的 dispose 是幂等的：

```ts id="dqqrva"
let disposed = false

return () => {
  if (disposed) return
  disposed = true

  renderScope.stop()
  container.textContent = ''
}
```

测试覆盖了：

```txt id="j0c1dz"
1. dispose 清空 container
2. dispose 后 effect 不再更新 DOM
3. dispose 可重复调用
4. render 前清理旧 container 内容
```

这个语义可以作为 MVP 的稳定行为。

# 需要注意的问题

## P1：`resetDelegatedEvents` 被作为 runtime-dom public API 导出了

`packages/runtime-dom/src/index.ts` 现在导出了：

```ts id="vutqk8"
export { bindEvent, delegateEvents, resetDelegatedEvents } from './events'
```

`resetDelegatedEvents()` 本质上更像 **test-only / internal API**。如果从 runtime-dom 主入口导出，后面它就会变成“半公开 API”，用户可能依赖它。更麻烦的是：它会删除全局 document 上所有 Zeus delegated listeners，生产环境用户误调用会导致全局事件委托失效。

建议改法：

```ts id="p5vsc9"
export { bindEvent, delegateEvents } from './events'
```

然后测试里不要从 `../src` 导入 `resetDelegatedEvents`，改成从内部路径导入：

```ts id="2v6h6b"
import { bindEvent, delegateEvents } from '../src'
import { resetDelegatedEvents } from '../src/events'
```

或者名字上明确 internal：

```ts id="jzddyg"
export function __resetDelegatedEventsForTest(): void {}
```

我的建议是：**不要从 runtime-dom public barrel 导出它**。

---

## P2：`delegatedListeners` 里的 `name` 暂时没用

现在类型是：

```ts id="g7jrw6"
type DelegatedListenerEntry = {
  name: string
  delegatedName: string
}
```

但 `resetDelegatedEvents` 只用 `delegatedName`。

可以简化成：

```ts id="hh487a"
const delegatedListeners: string[] = []
```

或者保留 `name`，但加注释说明未来用于 debug/devtools。这个不是 blocker。

---

## P2：`focusin` / `focusout` 直接绑定不是当前语义

现在 `focusin` 会被 `normalizeOriginalEventName('focusin')` 映射成 `focus`。

所以如果未来用户写：

```tsx id="0sehb7"
<div onFocusIn={...} />
```

并且 compiler 注册 `focusin`，runtime 会查 `__zeusEvents.focus`，不会查 `focusin`。当前 JSX 类型里没有 `onFocusIn`，所以不是问题；但后面如果要支持更多 DOM 原生事件名，需要重看这个映射策略。

建议在文档中只承诺：

```txt id="gj1kxd"
onFocus -> focusin
onBlur  -> focusout
```

不要暂时承诺 `onFocusIn/onFocusOut`。
