# 仍建议修的小问题

## 小问题 1：release 当前版本源仍优先读 root version

`release.ts` 现在还是：

```ts
const currentVersion =
  rootPkg.version ||
  JSON.parse(
    fs.readFileSync(
      path.resolve(__dirname, '../../packages/zeus/package.json'),
      'utf-8',
    ),
  ).version
```

文档里写的是“根 package.json 的 version 不作为真实版本来源”，但代码上还是优先读 root。这个不一定会马上出问题，因为 release 过程中 root version 会被同步更新；但从语义上建议改成 **直接以 `packages/zeus/package.json` 为准**：

```ts
const corePkgPath = path.resolve(__dirname, '../../packages/zeus/package.json')
const currentVersion = JSON.parse(fs.readFileSync(corePkgPath, 'utf-8')).version
```

这个属于语义修复，不是阻断。

## 小问题 2：publish dist-tag 仍然用全局 version 判断

`publishPackages(version)` 里 dist-tag 是根据传进来的全局 version 判断：

```ts
if (args.tag) releaseTag = args.tag
else if (version.includes('alpha')) releaseTag = 'alpha'
else if (version.includes('beta')) releaseTag = 'beta'
else if (version.includes('rc')) releaseTag = 'rc'
```

然后所有包共用这个 `releaseTag`。

在当前阶段 core 和 vite-plugin 都还是 `0.0.2`，问题不大。但未来如果出现：

```txt
@zeus-js/zeus@0.4.0
@zeus-js/vite-plugin@0.2.0-beta.1
```

就可能 dist-tag 判断不准确。

更稳的是按每个包自己的版本判断：

```ts
function resolveReleaseTag(version: string): string | null {
  if (args.tag) return args.tag
  if (version.includes('alpha')) return 'alpha'
  if (version.includes('beta')) return 'beta'
  if (version.includes('rc')) return 'rc'
  return null
}
```

然后在循环里：

```ts
const pkgVersion = pkg.packageJson.version as string
await publishPackage(
  pkg.name,
  pkgVersion,
  resolveReleaseTag(pkgVersion),
  additionalFlags,
)
```

这个也不是当前合并阻断，但建议后续修。

## 小问题 3：Release workflow 的 `--skipBuild` 依赖 precheck ✅ 已修

release workflow 先：

```yaml
- name: Release precheck
  run: pnpm release:precheck
```

再：

```yaml
run: pnpm release --publishOnly ${{ env.VERSION }} --skipBuild
```

这个现在可以工作，因为 precheck 会先 build。但它隐含了一个约定：**发布依赖 precheck 产物**。建议加注释，或者干脆发布阶段不 skip build，避免以后别人改 precheck 时把发布弄坏。
