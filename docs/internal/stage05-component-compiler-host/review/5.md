# 主要遗留问题：transform 作用域过宽

这是我认为 **发布 0.1.0 前最应该修的点**。

当前 `rollup.ts` 里：

```ts
const shouldTransform = createFilter({})
```

也就是说 transform 阶段没有使用 `components.include / exclude`，而是默认处理所有 `.ts/.tsx/.jsx` 文件。

`createFilter()` 默认 include 是：

```ts
;/\.[tj]sx(?:\?.*)?$/
```

默认只排除 `node_modules`。

而 `transformZeus()` 会直接跑 Zeus compiler。
compiler visitor 又会无条件处理 `JSXElement / JSXFragment`。

这会带来一个隐藏风险：

```txt
components.include 只分析 src/components/**/*
但 transform 可能会编译 App.tsx / demo.tsx / 业务 React JSX
```

比如 `examples/react-wrapper/src/App.tsx` 是正常 React JSX，但在当前 transform filter 下也有被 Zeus compiler 处理的风险。

CI build 能过，不代表浏览器运行时一定没问题，因为 React App 可能被编译成 Zeus DOM runtime 产物后再交给 React render。

---

## 建议修复方式

把 transform filter 和 analyzer 的 include/exclude 对齐。

### 推荐改法

`rollup.ts` 不要在外层固定：

```ts
const shouldTransform = createFilter({})
```

而是在 `buildStart()` 里基于 resolved root/include/exclude 生成 filter：

```ts
let shouldTransform = (_id: string) => false
```

```ts
async buildStart() {
  const root = resolveRoot(options.root)
  const include = resolveComponentInclude(options.components?.include)
  const exclude = resolveComponentExclude(options.components?.exclude)

  shouldTransform = createComponentTransformFilter({
    root,
    include,
    exclude,
  })

  // analyzer...
}
```

新增一个 filter：

```ts
import path from 'node:path'
import picomatch from 'picomatch'
import { cleanUrl } from './filter'

export function createComponentTransformFilter(options: {
  root: string
  include: string[]
  exclude: string[]
}) {
  const isIncluded = picomatch(options.include)
  const isExcluded = picomatch(options.exclude)

  return function shouldTransform(id: string): boolean {
    const clean = normalizePath(cleanUrl(id))
    const relative = normalizePath(path.relative(options.root, clean))

    if (relative.startsWith('..')) return false
    if (path.isAbsolute(relative)) return false
    if (isExcluded(relative)) return false

    return isIncluded(relative)
  }
}

function normalizePath(value: string): string {
  return value.replace(/\\/g, '/')
}
```

然后：

```ts
async transform(code, id) {
  if (!shouldTransform(id)) {
    return null
  }

  return await transformZeus({
    id,
    code,
    compiler: options.compiler,
    sourcemap: true,
  })
}
```

这样用户写：

```ts
zeus({
  components: {
    include: ['src/components/**/*.{ts,tsx}'],
  },
})
```

就只编译组件源码，不会误编译 `src/App.tsx`。
