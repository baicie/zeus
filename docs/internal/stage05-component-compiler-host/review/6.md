# 这次已解决的关键问题

## 1. transform 作用域已收窄

之前 `rollup.ts` 是全局处理 `.ts/.tsx/.jsx`，可能误编译 React/Vue consumer app。现在已经改成：

```ts
let shouldTransform = (_id: string) => false
```

并在 `buildStart()` 中根据组件 include/exclude 创建过滤器：

```ts
shouldTransform = createComponentTransformFilter({
  root,
  include,
  exclude,
})
```

transform 阶段也只在 `shouldTransform(id)` 命中时才跑 Zeus compiler。

这意味着：

```txt
components.include = ['src/components/**/*.{ts,tsx}']
```

时，只会编译组件源码，不会再误编译：

```txt
src/App.tsx
examples/react-wrapper/src/App.tsx
examples/vue-wrapper app 入口
普通 React/Vue demo 文件
```

这是发布前很重要的一步。

---

## 2. picomatch 依赖已补

新增 filter 使用了 `picomatch`，并且 `@zeus-js/bundler-plugin` 已经把它加入 dependencies。

这个没问题。

---

## 3. 新 filter 实现整体合理

`componentTransformFilter` 会：

```txt
1. 清理 query/hash
2. 转成相对 root 的路径
3. 排除 root 外文件
4. 先匹配 exclude
5. 再匹配 include
```

实现上是合理的。

这比之前的默认正则：

```ts
;/\.[tj]sx(?:\?.*)?$/
```

安全很多。

import path from 'node:path'

describe('componentTransformFilter', () => {
